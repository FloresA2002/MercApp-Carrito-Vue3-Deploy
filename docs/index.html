MercApp

Micrositio técnico del proyecto SPA desarrollado con Vue 3, API REST propia, MongoDB Atlas, Railway, Netlify y GitHub Pages.

Resumen técnico

MercApp es una aplicación web de una sola página desarrollada con Vue 3, Composition API y Vue Router. Consume un API REST propio construido con Node.js y Express. La información de productos y categorías se almacena en MongoDB Atlas y el despliegue se realizó usando Railway para el backend y Netlify para el frontend.

Arquitectura de despliegue

  • Frontend: Vue 3 + Vite desplegado en Netlify.
  • Backend: Node.js + Express desplegado en Railway.
  • Base de datos: MongoDB Atlas.
  • Micrositio: GitHub Pages desde la carpeta /docs.
  • CORS: configurado para permitir el dominio público de Netlify.
  • Variables de entorno: usadas para API URL y conexión MongoDB.

Enlaces del proyecto

Guía breve de ejecución local

Backend

Frontend

cd frontend

Endpoints principales

Método Ruta Descripción Ejemplo
GET /health Verifica que la API esté activa. { "status": "ok" }
GET /api/products Obtiene todos los productos. [{ "name": "Laptop Gamer" }]
GET /api/products/:id Obtiene un producto por ID. { "id": "...", "name": "Mouse RGB" }
POST /api/products Crea un nuevo producto. { "name": "...", "price": 20 }
PUT /api/products/:id Actualiza un producto completo. { "name": "Producto editado" }
PATCH /api/products/:id Actualiza parcialmente un producto. { "stock": 10 }
DELETE /api/products/:id Elimina un producto. { "message": "Producto eliminado" }
GET /api/categories Obtiene todas las categorías. [{ "name": "Electrónica" }]

Funcionalidades implementadas

  • Catálogo de productos.
  • Búsqueda por nombre y descripción.
  • Filtro por categoría.
  • Detalle de producto por ruta dinámica.
  • Carrito con localStorage.
  • CRUD de productos.
  • Lazy loading y Suspense.
  • Consumo de API mediante variables de entorno.
  • Backend conectado a MongoDB Atlas.
  • Despliegue con HTTPS en Railway, Netlify y GitHub Pages.
